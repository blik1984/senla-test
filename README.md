веб-приложение автоматической корректировки текста — Error Free Text. Система получает запрос с текстом от пользователя, в ответ отдает текст с исправленным опечатками.

Архитектура:
Монолитное REST приложение 

Стек технологий:
Язык: Java 17+
Фреймворк: Spring boot 3
Система сборки: Gradle
База данных: PostgreSQL
Контейнеризация: Docker (docker-compose)

Запросы:
1. Создание задачи на корректировку текста.
Запрос в теле принимает текст, который подлежит корректировке, вместе с параметром языка. Полученные данные сохраняются в базу данных со статусом новой задачи, а в ответе отдается идентификатор созданной задачи.
2. Получение исправленного текста по идентификатору задачи.
Если задача уже завершена — пользователю возвращается ответ, содержащий соответствующий статус, а также скорректированный текст.
Если задача находится еще в процессе — ответ содержит только соответствующий статус.
Если выполнение задачи завершилось с ошибкой — возвращается соответствующий статус с описанием ошибки.

Логика корректировки текста:
В приложении работает Scheduler. С заданной частотой он достает из БД данные по задачам, которые необходимо обработать и, используя стороннее API, корректирует текст. Скорректированный текст сохраняется в базу данных с обновлением статуса задачи.
Для корректировки текста использовать API Яндекса метод POST checkTexts:
По документации API Яндекса принимает запросы в 10 000 символов. Наше приложение должно принимать текст большего размера и корректировать его частями.
По документации API Яндекса принимает параметр options. Наше приложение перед отправкой запроса/ов на API Яндекса должно определить:
Если в тексте есть цифры — включить опцию IGNORE_DIGITS.
Если текст содержит URL — включить опцию IGNORE_URLS.
Настройки FIND_REPEAT_WORDS и IGNORE_CAPITALIZATION всегда выключены.

Если при обращении к API Яндекса произошла ошибка, необходимо установить соответствующий статус у задачи и прекратить ее обработку.


Требования к реализации:
Входящие запросы должны валидироваться: 
корректный параметр Языка (EN или RU),
текст должен содержать минимум 3 символа,
текст не может содержать только спецсимволы и цифры.
Приложение должно быть написано с учетом принципов REST.
Должно присутствовать логирование и обработка ошибок.
При возникновении ошибок обработки запросов ответ должен содержать информацию об ошибке:
{
	"errorMessage": "Task with id: 44bd78dc-d08c-41c6-b87d-fb82046bd470 not found",
	"errorCode": 40401,
	"timestamp": "2005-08-09T18:31:42.201',
	"path": "/tasks"
}
Юнит-тесты (обязательно), интеграционные (опционально).
Код должен соответствовать принципам Clean Code, Java style.
